# Mini Inventory API / API Mini Inventario

## English

Mini Inventory API is a REST API built with FastAPI and MongoDB for managing products and stock movements. It supports product CRUD operations, product filtering, pagination, stock entries, stock exits, and automatic stock validation before outgoing movements.

### Features

- Product creation with unique SKU validation.
- Product listing with filters by name, category, and price range.
- Pagination with `skip` and `limit`.
- Product lookup by MongoDB ObjectId.
- Product updates for name, category, and price.
- Product deletion only when stock is `0`.
- Stock movement registration for entries (`in`) and exits (`out`).
- Stock validation to prevent outgoing movements greater than available stock.
- Swagger UI and ReDoc documentation generated by FastAPI.
- Health check endpoint.

### Tech Stack

### API Endpoints

#### General

| Method | Endpoint | Description |
| --- | --- | --- |
| GET | `/` | API welcome message and endpoint summary |
| GET | `/health` | Service health check |

#### Products

| Method | Endpoint | Description |
| --- | --- | --- |
| POST | `/api/products/` | Create a product |
| GET | `/api/products/` | List products with filters and pagination |
| GET | `/api/products/{product_id}` | Get one product by ID |
| PATCH | `/api/products/{product_id}` | Update a product |
| DELETE | `/api/products/{product_id}` | Delete a product only if stock is `0` |

Product creation body:

```json
{
  "sku": "SKU-001",
  "name": "Wireless Mouse",
  "category": "Electronics",
  "price": 25.99,
  "stock": 10
}
```

Product update body:

```json
{
  "name": "Wireless Mouse Pro",
  "category": "Electronics",
  "price": 29.99
}
```

Product list query parameters:

| Parameter | Type | Description |
| --- | --- | --- |
| `name` | string | Case-insensitive name search |
| `category` | string | Exact category filter |
| `skip` | integer | Number of items to skip |
| `limit` | integer | Number of items to return, from 1 to 100 |
| `price_min` | number | Minimum price |
| `price_max` | number | Maximum price |

Example:

```bash
curl "http://127.0.0.1:8000/api/products/?category=Electronics&price_min=10&price_max=100"
```

#### Movements

| Method | Endpoint | Description |
| --- | --- | --- |
| POST | `/api/movements/` | Register a stock movement |
| GET | `/api/movements/{product_id}` | List movements for a product |

Movement creation body:

```json
{
  "product_id": "665f1f7c2a5b8a001234abcd",
  "type": "in",
  "qty": 5
}
```

Movement types:

| Type | Description |
| --- | --- |
| `in` | Adds stock to the product |
| `out` | Removes stock from the product if enough stock is available |

Movement list query parameters:

| Parameter | Type | Description |
| --- | --- | --- |
| `start_date` | string | Optional start date in `YYYY-MM-DD` format |
| `end_date` | string | Optional end date in `YYYY-MM-DD` format |

Example:

```bash
curl "http://127.0.0.1:8000/api/movements/665f1f7c2a5b8a001234abcd?start_date=2025-01-01&end_date=2025-12-31"
```

### Example Workflow

1. Create a product:

```bash
curl -X POST "http://127.0.0.1:8000/api/products/" \
  -H "Content-Type: application/json" \
  -d "{\"sku\":\"SKU-001\",\"name\":\"Wireless Mouse\",\"category\":\"Electronics\",\"price\":25.99,\"stock\":10}"
```

2. Add stock:

```bash
curl -X POST "http://127.0.0.1:8000/api/movements/" \
  -H "Content-Type: application/json" \
  -d "{\"product_id\":\"PRODUCT_ID_HERE\",\"type\":\"in\",\"qty\":5}"
```

3. Remove stock:

```bash
curl -X POST "http://127.0.0.1:8000/api/movements/" \
  -H "Content-Type: application/json" \
  -d "{\"product_id\":\"PRODUCT_ID_HERE\",\"type\":\"out\",\"qty\":3}"
```

### Notes

- MongoDB indexes are created on startup when the database user has permission.
- The `sku` field is indexed as unique.
- Products with stock different from `0` cannot be deleted.
- Outgoing movements fail when the requested quantity is greater than the current stock.
- Keep `.env` out of version control because it can contain database credentials.
